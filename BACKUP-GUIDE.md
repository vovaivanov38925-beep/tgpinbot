# 🛡️ Система бекапов tgpinbot

## 📁 Файлы

| Файл | Описание |
|------|----------|
| `backup-create.bat` | Создать бекап перед изменениями |
| `backup-restore.bat` | Откатиться к бекапу |
| `backup-list.bat` | Показать список всех бекапов |
| `release-create.bat` | Создать стабильный релиз (v1.0.0) |

## 🚀 Рекомендуемый рабочий процесс

### Перед большими изменениями:
```
1. Запусти backup-create.bat
2. Опиши что собираешься делать
3. Работай спокойно
```

### Если что-то сломалось:
```
1. Запусти backup-restore.bat
2. Выбери последний рабочий бекап
3. Продолжай работу
```

### Когда всё работает отлично:
```
1. Запусти release-create.bat
2. Укажи версию (1.0.0, 1.1.0, 2.0.0)
3. Это будет стабильная точка возврата
```

## 💡 Примеры использования

### Создать бекап:
```batch
backup-create.bat
# Ввод: "перед добавлением системы оплаты"
# Результат:
#   - Тег: v-backup-20250115-1430
#   - Ветка: backup/backup-20250115-1430
```

### Откатиться:
```batch
backup-restore.bat
# Выбираешь нужный бекап из списка
# Текущее состояние сохраняется в emergency-ветку
```

### Посмотреть бекапы:
```batch
backup-list.bat
# Показывает все теги и ветки бекапов
```

## ⚠️ Важно!

- **Всегда создавай бекап перед большими изменениями**
- **Проверяй работу после отката**
- **Emergency-ветки сохраняют сломанное состояние на случай**

## 🔄 Альтернативные команды (ручные)

### Git команды для бекапа:
```bash
# Создать тег
git tag -a v-backup-20250115 -m "описание"
git push --tags

# Создать ветку бекапа
git branch backup/перед-изменениями
git push origin backup/перед-изменениями

# Откатиться к тегу
git checkout v-backup-20250115

# Принудительно вернуть master к тегу
git checkout v-backup-20250115
git checkout -B master
git push -f origin master
```
