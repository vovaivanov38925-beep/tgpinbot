## Правила бэкапов

1. **Перед крупными изменениями** - создавать git tag: `vX.Y.Z-description`
2. **После каждого изменения** - обновлять CHANGELOG.md
3. **Теги:**
   - `stable` - стабильная рабочая версия
   - `fix-*` - исправления багов
   - `feat-*` - новый функционал

### Команды для бэкапа:
```bash
# Создать тег
git tag -a "v1.0.1-fix-tabs" -m "Описание изменений"

# Посмотреть все теги
git tag -l

# Откатиться к тегу
git checkout v1.0.0-stable
```
